  3. [Discuss why we use encapsulation in classes. Why do we want to "hide" code within our class? Explain how we can create "private" and "public" members in a constructor function.]

    4. [Using encapsulations helps to insure that the object's internal state is protected and can not be directly tampered with. Public members are accessible from outside the class, and can be directly modified. Private members are hidden from the outside, and can only be accessed or modified through specific public methods or internal logic.]

  4. [Explain what happens when we use the "new" command in JavaScript.]

    5. [Using new allows you to create multiple instances of the same type, each having its own unique set of properties and methods, but sharing methods defined on the constructor's prototype. This is key to enabling object-oriented patterns in JavaScript.]
